# Dotline — принятые решения (лог)

Зачем файл: чтобы потом было понятно **что и почему** решено. Каждая запись —
решение + причина. Свежие сверху.

---

## 2026-07-08 — большая UI-сессия

### Цвета: два акцента, разведены намеренно
- **`--line` зелёный** — только для таймлайна (точки, линии, стрелки, часы,
  connector-точка, «+», превью карточек).
- **`--accent` фиолетовый** — весь UI (кнопки, Apply, фокус-рамки, слайдер,
  sync-dot, аватар, hover карточек, звезда хайлайта, армированные подсказки драга).
- **Почему:** пользователь хотел, чтобы «нить» осталась зелёной, а интерфейс был
  другого цвета; смешение читалось как мусор.

### Шрифт: округлый, но БЕЗ CJK в стеке
- `--sans: ui-rounded, "SF Pro Rounded", system-ui, …` — **никаких CJK-шрифтов**
  (Hiragino Maru Gothic и т.п.).
- **Почему:** SF Pro Rounded не покрывает всю кириллицу → браузер брал японский
  шрифт из стека и рендерил русский текст «разряженным». Падаем сразу на
  system-ui (SF Pro) — кириллица нормальная. Округлость для мелкого текста
  необязательна (решение пользователя). Nunito self-host рассматривали и
  отклонили как избыточный.

### Типографика: шкала из токенов
- 7 ступеней `--fs-2xs…--fs-xl` (+ `--fs-display` для геро-значения модалок),
  веса `--w-regular/medium/semibold/bold`. **Почему:** было 16 ad-hoc размеров и
  6 весов — «взрыв»; свели к системе.

### Стрелки на линии — чистая геометрия, БЕЗ цветовых масок
- Правило стрелки: линия с треугольником НА КОНЦЕ, кружок чистый (без разрезов и
  хвостов), зазор ~5px между кружком и стрелкой.
- **Реализация:** укорачиваем рельсу (`.link.arr-*::before` bottom/top: 9px),
  треугольник — `.link.arr-*::after` на конце, а «пенёк» рельсы у целевого
  кружка глушим классом `.no-stub-top/.no-stub-bottom` (ставится в
  `applyArrowStubs`). Кружок непрозрачный → всегда чистый круг.
- **Почему так:** прежний подход (маска цветом фона поверх рельсы) был хрупким —
  залезал на кружок (белые «разрезы»/notch) и оставлял хвосты, ломался в
  тёмной/светлой теме и на часах-хайлайте.
- Направление следует за временем/типом: `down` = следующая точка позже, `up` =
  раньше. В reverse — посегментно (в т.ч. смена направления в середине).

### Драг точки: две стороны + подсказки
- **Влево → Delete (красная подсказка), вправо → Add** (пользователь уточнил
  именно так). Слабое/случайное движение — ничего; тап = «выполнено».
- Порог: **touch 30px**, мышь 80px. **Почему:** на мобиле большой порог требовал
  тянуть за экран.
- При старте драга снимаем фокус с поля → **на мобиле не всплывает клавиатура**.
- Подписи точки скрываются во время драга (не мешались с подсказками).

### Highlight
- Маленькая **☆ слева** от точки; тап → **★**, точка выделяется (крупнее).
- **Тап «выполнено» гасит хайлайт** («сделал и хорошо»).
- (В работе) редизайн: текст хайлайта через весь экран по центру над точкой,
  линия прячется под надписью; в тёмной теме подложка тёмно-серая, не белая.

### Модалки времени — не ломать модуль
- **Add event оставили полным** (дата + часы + ползунок ±6ч). Был откат: я
  подменил его на интервал-версию ради «дефолт −30», пользователь справедливо
  возмутился. Дефолт −30 и так был (`previousTime − 30 мин`).
- Три модалки — один паттерн: крупное гер-значение = что задаёшь, мелко снизу =
  следствие; без заголовков; крестик + клик-вне; круглый Apply с галочкой;
  frosted-glass фон.

### Ввод текста
- **Enter = перенос строки** внутри названия пункта (не новый пункт).
  **Shift+Enter** — то же. Добавлять пункты теперь: «+» снизу или драг вправо.

### Главный экран
- Секции **Dotline** и **Timed dotline**; вместо «заголовок+подпись+add-карточка»
  — компактная строка **«⊕ Название»** (тап по кружку или слову = новая заметка).
- Карточки — горизонтальная лента; новые слева (unshift). *(Открытые правки:
  свайп-пейджинг со снапом по центру и «ушками» по бокам; сейчас обычный скролл.)*

### Нумерация заметок
- Стабильный `autoNum` на заметке (не по позиции). **Почему:** позиционная
  нумерация + unshift давали всем «Timing 1».

### Прочее
- Reverse-раздел переименован: `Timing Dotlines` → в UI «Timed dotline»
  (внутренний `type: "reverse"` не трогаем — совместимость).
- «Plan Check» вынесен в отдельный `PLAN_CHECK_SPEC.md` (фичи ещё нет).

## 2026-07-09 — шаринг, real-time, цвет карточек, карусель

### Шаринг заметок (снапшот)
- Кнопка **Share** в топбаре (правый край). Создаёт **неизменяемый снапшот**
  заметки на сервере (таблица `shares`), токен ~12 симв.
- **Две ссылки** в модалке: главная — интерактивная `?s=<token>` (read-only
  страница в приложении); мелкая — **«Ссылка для твоего ИИ»** =
  `/api/shared/<token>/text` (`text/plain`). **Почему текстовая:** SPA рендерит
  на клиенте, fetch (ИИ/превью) видит только каркас; текст клиент кладёт сам
  (он знает часовой пояс → время корректное).
- Read-only режим (`viewOnly`): в общей заметке отключены редактирование, драг,
  «+», пикеры, звёзды; back → Close; баннер; **Make a copy** клонирует к себе.
- **Известный риск:** `/api/share` без авторизации и без rate-limit — любой
  может неограниченно плодить снапшоты (рост БД/диска). Для v1 ок; при публичном
  росте добавить лимит/чистку старых.

### Real-time режим (timing-заметки)
- Кнопка **◎/◉ Real Time** (левый край топбара, только reverse). При включении:
  высота сегмента **пропорциональна длительности** (~1.5px/мин, floor 46, cap
  380) → 1ч короче 2ч; и **пунктирная линия «now»** на позиции текущего времени
  (интерполяция между точками), тик раз в 30с, прячется если «сейчас» вне
  диапазона плана.
- Линия «now» **за текстом** (z-index 1 < узлов) и с горизонтальным градиентом
  (к центру → почти цвет фона), чтобы не зачёркивать подписи. Дэши через `::before`
  + mask (иначе маска резала бы подпись «now»).

### Цвет карточек
- На карточке снизу-справа: кнопка-палитра (градиент) = случайный цвет из
  **согласованного сета** (16 цветов, каждый с парным читаемым текстом AAA/AA);
  повторный тап — следующий; ниже контурный кружок — сброс. Полоски/точки превью
  и текст берут парный тон. Пустой кружок превью заливается **цветом карточки**
  (иначе линия просвечивала сквозь него).

### Главная = карусель
- Секции **Dotline** / **Timed dotline**: «⊕ Название» (тап = новая), заметки —
  горизонтальный **свайп-пейджинг** (snap по центру + `scroll-snap-stop`, «ушки»
  по бокам). Новые карточки слева. Мобайл: карточки 70%, 1.4:1.

### Прочие фиксы
- **Кэш:** nginx отдаёт HTML заметок с `Cache-Control: no-store` — правки больше
  не залипают в кэше браузера (весь день была эта боль).
- **Дата-пикер:** нативный календарь открывается через `dateInput.showPicker()`
  по клику (прозрачный `<input type=date>` в Chrome сам не открывается).
- **out-of-order:** флаг ставится на саму точку-разворот (сегмент ДО и ПОСЛЕ
  разного знака), а не на следующую (был off-by-one).
- **Enter/Shift+Enter** = перенос строки в названии (оба).
- **nowTimer**: чистится в начале `drawReverseSpine` (не только в render), иначе
  накапливались setInterval при правках realtime-заметки.

### UI-ревью timeline: центрирование на мобиле и gap у now-line
- **Мобильный reverse timeline:** точка переезжает в центр экрана, время
  располагается над точкой, название события — под ней; обе строки выровнены
  по центру. Рельса и стрелки перестраиваются под центральную точку.
  **Почему:** на узком экране время+название «уезжали» вправо от центра.
- **Now-line не зачёркивает время/дату:** у `.node-alias` и `.node-label`
  добавлен псевдоэлемент с фоном `--surface`, который создаёт широкий
  вертикальный разрыв поверх пунктирной линии «now». Разрыв длиннее по
  вертикали, чем прежний градиентный fade.
  **Почему:** линия «now» залезала на текст времени и даты, особенно когда
  текущее время совпадало с узлом.

## Открытая очередь (на следующий заход)
- Шаринг: rate-limit / чистка старых снапшотов, если сервис станет публичным.
- (Опц.) server-side превью для `?s=` ссылки — красивые карточки в мессенджерах.
- Real-time: пересчёт позиции линии «now» на resize (сейчас раз в 30с/на ререндер).
